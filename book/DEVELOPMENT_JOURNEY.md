## ðŸŽ­ Model Selection Guide

### Use OPUS for:
- **Architecture Design**: System design, database schema, API structure
- **Complex Algorithms**: Pattern detection, learning loops, prediction models
- **Security Design**: Encryption, access control, audit systems
- **Integration Strategy**: Multi-system coordination, protocol design
- **Optimization Logic**: Performance algorithms, caching strategies
- **Advanced Features**: AI behaviors, predictive systems

### Use SONNET for:
- **Implementation Code**: Writing actual code from specs
- **API Integration**: Connecting to external services
- **UI/UX Development**: Dashboards, interfaces
- **Configuration Files**: Docker, deployment scripts
- **Testing**: Unit tests, integration tests
- **Documentation**: Code comments, API docs
- **Bug Fixes**: Debugging, error handling
- **Routine Features**: CRUD operations, basic workflows

### General Rule:
```
IF (creative_problem_solving || system_design || complex_logic)
  USE Opus
ELSE IF (code_implementation || standard_patterns || known_solutions)  
  USE Sonnet
```
